01 · Architecture

Three beats, one spec, zero fabrication

BEAT 1EstimatorVoice interview (ElevenLabs widget) + doc upload (rent agreement / requirements). Both paths emit the same JSON job spec. User confirms before any call.
BEAT 2CallerNegotiator agent calls each dealer (counter-agent or role-play human). Identical spec pitch every call. Logs itemised quotes mid-call via tool webhook.
BEAT 3CloserSecond-round calls with leverage from logged bids. Red-flag engine screens quotes. Report ranks all quotes with transcript citations.

Services

ServiceHostResponsibilities
Next.js frontendVercel freePages: Dashboard · Intake (voice widget + upload) · Spec Confirm · Call Center (live/monitor) · Report. ElevenLabs React SDK for all browser audio.
FastAPI backendRender free + pingerDoc parsing (OpenAI), benchmark service (Tavily + cache), spec/quote CRUD, agent tool webhooks, agent-to-agent audio bridge, report generation.

Agent-to-agent bridge (highest-risk component)

FastAPI asyncio task opens two ElevenLabs conversation WebSockets — Negotiator and a Dealer persona — and cross-pipes audio: Negotiator output → Dealer input, Dealer output → Negotiator input. The recording is a time-aligned stereo WAV (left = negotiator, right = dealer — the two voices are never summed into one channel, so they replay in conversation order without overlapping), and both legs are live-streamed to the browser as separate leg-tagged streams so judges hear it live. Transcripts arrive on both sockets as events; store both sides.

Guaranteed fallback If the bridge fights back past its 3-hour timebox: role-play mode. A teammate opens the Dealer page in a second browser tab/laptop and answers the Negotiator's call by mic, playing the persona from a one-page character card. Needs zero extra code (same widget as intake) and the brief explicitly allows it. The demo cannot die on this component.

The honesty guardrail — say this in the pitch

The Negotiator has no free-text knowledge of other bids. Its only source of leverage is the get_leverage tool, which returns real quotes logged in Supabase during earlier calls. If no bid exists, the tool returns nothing and the prompt instructs the agent to negotiate on fees and terms instead. Fabricating a competing bid is not a prompt hope — it is architecturally impossible. Judges score honesty constraints explicitly; this is the strongest possible answer.

02 · Data model

Supabase schema

-- keep it this small; resist adding tables at 4 AM
specs      (id, created_at, vertical, status, spec_json jsonb, benchmark_json jsonb, confirmed bool, user_id uuid)
dealers    (id, spec_id, name, persona, phone_label, source)          -- persona: stonewaller|lowballer|upseller|firm|human
calls      (id, spec_id, dealer_id, round int, status, started_at, ended_at,
            recording_url, transcript_json jsonb, outcome)             -- outcome: quote|callback|declined
quotes     (id, call_id, dealer_id, monthly_rent int, advance_months int, commission int,
            maintenance int, annual_increment_pct numeric, other_fees jsonb,
            total_first_year int,  -- computed, the comparison key; excludes annual_increment_pct (year-2+ effect)
            binding bool, notes, flagged bool, flag_reason)
  • total_first_year = 12×rent + advance + commission + 12×maintenance + other fees. Single number makes deliberately-incomparable fee structures comparable — that is the product thesis in one column.
  • transcript_json stores numbered utterances → report cites [call 3, line 41].
  • Frontend reads via FastAPI, not Supabase directly — keeps keys server-side, one auth story.
  • specs.user_id ties every spec to a Supabase Auth user — FastAPI enforces ownership on every read/write, closing the pre-multi-tenant cross-tenant leak.

04 · The cast

Six agents, one factory script

IntakeThe Estimator

Interviews like a pro estimator: fills every spec_schema field, ≤6 questions if doc upload pre-filled most. Confirms parsed rent-agreement data verbally ("your current shop is 400 sqft at 85k — correct?"). Emits final spec via tool call.

Caller / CloserThe Negotiator

Pitches spec identically each call. Extracts every fee in fee_taxonomy — refuses vague ranges, asks "what's included in that?". Round 2: pulls get_leverage, pushes price-match, questions fees, ends every call with structured outcome via log_quote.

Dealer 1The Stonewaller

"Rates depend, sahib. Come to the office." Gives numbers only under persistent, specific questioning. Tests extraction stamina. Concedes a range → exact figure when spec is detailed.

Dealer 2The Lowballer

Headline rent 35–45% under market. Advance, commission, "society maintenance" surface only when asked. The red-flag system's demo target. Never volunteers total cost.

Dealer 3The Upseller

Pushes the bigger corner shop, "another party is viewing tomorrow", pressure tactics. Tests spec discipline — Negotiator must decline the upsell and return to the confirmed spec.

Dealer 4Firm-but-Fair

Quotes a realistic price, holds it — until presented with a genuine logged competing bid, then matches or trims 5–8% and waives commission. This call is the mandatory "price moves from leverage" success criterion.

The four dealer personas above simulate real property dealers for testing/demo — the SaaS product's real end-state calls actual dealer phone numbers (Twilio, a listed stretch item).

Negotiator tool belt (webhooks → FastAPI)

ToolDoes
log_quoteWrites itemised quote row mid-call. Validates against fee taxonomy; missing fee → agent instructed to ask again before hanging up.

10 · Edge

Six moves that outscore bigger builds

  1. The 30-second config swap is the cheapest large point-haul available. Rough second config is enough; the moment is what's scored.
  2. Caption the four mandatory moments. Judges grade checklists at speed; make every criterion impossible to miss.
  3. Transcript citations in the report ([call 3, line 41]) turn "trust us" into evidence — directly named in the success criteria.
  4. Say "cannot lie by architecture" out loud in the pitch. Most teams will answer honesty with "we prompted it not to." You have a structural answer.
  5. Keep intake ≤ 6 spoken questions. Doc parsing pre-fills; interview confirms. A 4-minute intake bores the video's first act.
  6. Golden calls by 4 AM. One recorded ugly-but-real end-to-end call is worth more than any feature added after it. It is the submission's insurance policy.
North star A connected loop beats a polished fragment — the brief says so verbatim. Every hour, ask: does the loop still run end to end? If yes, polish. If no, fix the loop. Nothing else matters.
